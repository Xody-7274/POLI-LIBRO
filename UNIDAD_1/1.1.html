1.1 Aspectos Básicos del Sistema Operativo

Estructura de un Sistema Operativo

Concepto:

"Un sistema operativo (SO) es un software que actúa como intermediario entre el hardware de una computadora y los programas de usuario, gestionando los recursos del sistema (como la CPU, memoria, almacenamiento y periféricos) y proporcionando servicios comunes para facilitar la ejecución de aplicaciones"
(Silberschatz, Galvin y Gagne, 2018, p. 3).
1.1.1
Objetivos

Aspectos básicos:

  1. Gestión de procesos: Controla la ejecución de múltiples programas y asigna tiempo de CPU.
  2. Gestión de memoria: Administra la memoria principal (RAM) y virtual.
  3. Sistema de archivos: Organiza y almacena datos en dispositivos de almacenamiento.
  4. Interfaz de usuario: Proporciona entornos gráficos (GUI) o de línea de comandos (CLI).
  5. Seguridad y acceso: Controla permisos y protege los recursos del sistema.

Abstracción proporcionada por el sistema operativo

Concepto:

El sistema operativo (SO) actúa como una capa de abstracción entre el hardware físico y las aplicaciones de usuario, simplificando la interacción con los recursos del sistema. Esta abstracción permite que los programas no necesiten gestionar directamente los detalles complejos del hardware, sino que interactúen con interfaces estandarizadas proporcionadas por el SO.

"El sistema operativo proporciona abstracciones de los recursos físicos, como procesadores, memoria, discos y dispositivos de E/S, para que los programas puedan operar de manera más simple y portátil"
(Tanenbaum & Bos, 2015, p. 4).

Principales abstracciones del SO:

  1. Procesos (Abstracción de la CPU):
    • El SO maneja la ejecución de programas como procesos, ocultando los detalles de la planificación y el multiplexado de la CPU.
  2. Espacio de direcciones (Abstracción de la memoria):
    • Proporciona la ilusión de memoria contigua y protegida mediante memoria virtual, independientemente de la distribución física de la RAM.
  3. Archivos (Abstracción del almacenamiento):
    • Organiza los datos en un sistema jerárquico de archivos, ocultando los detalles del manejo de sectores en discos.
  4. Dispositivos de E/S (Abstracción del hardware):
    • Ofrece interfaces genéricas (controladores) para interactuar con periféricos (teclados, impresoras, etc.) sin requerir conocimiento específico del hardware.
  5. Interfaz de usuario (Abstracción de la interacción):
    • Proporciona entornos gráficos (GUI) o de comandos (CLI) para que el usuario no necesite operar directamente con instrucciones de bajo nivel.

Administración de Recursos en Sistemas Operativos

Concepto:

El sistema operativo (SO) es el responsable de gestionar los recursos del hardware de manera eficiente, asegurando su correcta asignación entre múltiples procesos y usuarios.

"La principal tarea de un sistema operativo es administrar los recursos de una computadora, incluyendo el procesador, la memoria, los dispositivos de almacenamiento y los periféricos, de manera que se maximice el rendimiento y se eviten conflictos"
(Stallings, 2018, p. 27).

Recursos gestionados por el SO:

  1. Gestión del Procesador (CPU):
    • Asigna tiempo de CPU a los procesos mediante planificación (scheduling).
    • Implementa mecanismos como multiprogramación, multitarea y tiempo compartido.
  2. Gestión de Memoria (RAM y Memoria Virtual):
    • Controla la asignación y liberación de memoria para procesos.
    • Implementa técnicas como paginación y segmentación para optimizar su uso.
  3. Gestión de Almacenamiento (Discos y Sistemas de Archivos):
    • Administra el espacio en discos duros, SSDs y otros medios.
    • Proporciona una estructura lógica (sistema de archivos) para almacenar y recuperar datos.
  4. Gestión de Dispositivos de Entrada/Salida (E/S):
    • Controla el acceso a periféricos (impresoras, teclados, redes) mediante controladores (drivers).
    • Implementa buffering y spooling para mejorar la eficiencia.
  5. Gestión de Redes (Comunicación entre Sistemas):
    • Facilita la comunicación entre computadoras mediante protocolos de red.

Objetivos de la Administración de Recursos:

✔ Eficiencia: Maximizar el uso de los recursos disponibles.

✔ Equidad: Distribuir recursos de manera justa entre procesos.

✔ Aislamiento: Evitar que un proceso monopolice o dañe el sistema.

✔ Seguridad: Controlar el acceso a los recursos según permisos.

Aislamiento en Sistemas Operativos

Concepto:

El aislamiento en sistemas operativos se refiere a los mecanismos que garantizan que los procesos, usuarios o aplicaciones no interfieran entre sí ni con el núcleo del sistema, protegiendo la integridad, confidencialidad y disponibilidad de los recursos.

"El aislamiento es un principio fundamental en el diseño de sistemas operativos, que previene que fallos o ataques en un proceso afecten a otros o al sistema en general"
(Anderson, 2020, p. 145).

Mecanismos de Aislamiento:

  1. Aislamiento de Procesos:
    • Cada proceso ejecuta en su espacio de direcciones virtuales, evitando que acceda a la memoria de otros procesos.
    • Implementado mediante protección de memoria (MMU) y segmentación-paginación.
  2. Aislamiento de Usuarios:
    • Los sistemas multiusuario (ejemplo:: Linux, Windows) asignan permisos (UID/GID) para restringir el acceso a archivos y recursos.
  3. Aislamiento del Kernel (Modo Supervisor vs. Usuario):
    • El núcleo del SO ejecuta en modo kernel (anillo 0), mientras que los procesos lo hacen en modo usuario (anillo 3), limitando operaciones privilegiadas.
  4. Virtualización y Contenedores:
    • Máquinas virtuales (VMs) y contenedores (Docker) aíslan entornos completos o aplicaciones para evitar conflictos.
  5. Sandboxing:
    • Técnica que ejecuta aplicaciones en entornos restringidos (ejemplo: navegadores web, apps móviles).

Importancia del Aislamiento:

  • Seguridad: Previene exploits como buffer overflows o inyección de código.
  • Estabilidad: Evita que un proceso defectuoso colapse el sistema.
  • Privacidad: Protege datos sensibles entre usuarios o aplicaciones.

Ejemplos prácticos:

  • En Linux, los namespaces y cgroups aíslan recursos en contenedores.
  • Windows usa Virtual Trust Levels (VTLs) para aislamiento de procesos críticos.
1.1.2
Evolución de los Sistemas Operativos

Concepto:

Los sistemas operativos (SO) han evolucionado desde programas básicos para gestionar hardware hasta plataformas complejas que soportan multitarea, redes, seguridad avanzada y computación distribuida. Su desarrollo ha estado ligado al avance del hardware y las necesidades de los usuarios.

"La historia de los sistemas operativos refleja la transición desde el procesamiento en serie hasta los entornos interactivos, paralelos y en tiempo real, impulsados por las demandas de eficiencia y usabilidad"
(Tanenbaum & Bos, 2015, p. 32).
"La evolución de los sistemas operativos ha sido impulsada por la necesidad de abstraer el hardware, mejorar la eficiencia y adaptarse a nuevos paradigmas como la movilidad y la nube"
(Silberschatz, Galvin y Gagne, 2018, p. 45).

Línea del Tiempo: Evolución de los Sistemas Operativos

1. Década de 1940–1950: Sin Sistemas Operativos

  • 1945–1955: Computadoras como ENIAC y UNIVAC se programaban manualmente mediante interruptores y cables.
  • No existía SO: Los usuarios gestionaban directamente el hardware.

2. Década de 1950–1960: Sistemas Batch (Procesamiento por Lotes)

  • 1956: GM-NAA I/O (General Motors), considerado el primer SO primitivo, para ejecutar trabajos en secuencia.
  • 1961: OS/360 (IBM), diseñado para mainframes, introdujo manejo de archivos y dispositivos.

3. Década de 1960–1970: Multiprogramación y Tiempo Compartido

  • 1965: CTSS (MIT), primer sistema de tiempo compartido.
  • 1969: Nace UNIX (Bell Labs), base de SO modernos (Linux, macOS).

4. Década de 1980: Computadoras Personales e Interfaces Gráficas

  • 1981: MS-DOS (Microsoft), SO dominante en PCs.
  • 1984: Mac OS (Apple), primera GUI comercial exitosa.
  • 1985: Windows 1.0 (Microsoft), complemento gráfico para MS-DOS.

5. Década de 1990: Redes y SO Modernos

  • 1991: Linux (Linus Torvalds), kernel libre basado en UNIX.
  • 1993: Windows NT (Microsoft), enfoque en estabilidad y redes.
  • 1995: Windows 95, populariza interfaz gráfica y multitarea.

6. Siglo XXI: Móviles, Virtualización y Nube

  • 2001: Mac OS X (Apple), basado en UNIX (BSD).
  • 2007: iOS (Apple) y 2008: Android (Google), dominan el mercado móvil.
  • 2010s: Expansión de virtualización (VMware, KVM) y contenedores (Docker, Kubernetes).
  • 2020s: SO para IoT (Raspbian, FreeRTOS) y computación cuántica.

Temas: Procesamiento por Lotes, Sistemas en Lotes con Spool y Sistemas Multiprogramados

1. Procesamiento por Lotes (Batch Processing)

Definición:

Método donde los trabajos se agrupan en "lotes" y se ejecutan secuencialmente sin interacción del usuario.

Características:

  • Década de 1950–1960 (mainframes como IBM 1401).
  • Los trabajos se organizaban en tarjetas perforadas o cintas magnéticas.
  • Limitación: Ociosidad de la CPU durante operaciones de E/S.
"El procesamiento por lotes optimizaba el uso de costosos sistemas mainframe, pero carecía de interactividad"
(Tanenbaum y Bos, 2015, p. 34).

2. Sistemas en Lotes con Spool (Simultaneous Peripheral Operations Online)

Definición:

Técnica que supera la ociosidad de la CPU almacenando trabajos en discos (cola de impresión o ejecución) mientras otros procesos usan la CPU.

Funcionamiento:

  1. Los trabajos se cargan desde tarjetas a disco (spooling input).
  2. La CPU ejecuta un trabajo mientras otro se carga/descarga en paralelo.
  3. Resultados se guardan en disco (spooling output) para imprimirse después.

Ejemplo:

  • IBM OS/360 (1960s) usaba spooling para gestionar impresoras y lectores de tarjetas.
"El spooling permitió mayor solapamiento entre E/S y cómputo, reduciendo tiempos muertos"
(Stallings, 2018, p. 52).

3. Sistemas Multiprogramados

Definición:

SO que mantiene múltiples programas en memoria, alternando la CPU entre ellos cuando uno espera por E/S.

Ventajas:

  • Mayor utilización de la CPU: Evita ociosidad durante esperas de E/S.
  • Primer paso hacia la multitarea: Base para sistemas modernos.

Mecanismos clave:

  • Gestión de memoria: Asignación dinámica a múltiples procesos.
  • Planificación (scheduling): Algoritmos como FCFS o Round-Robin.

Ejemplo:

  • UNIX (1969) y OS/360 MFT (Multiprogramming with a Fixed number of Tasks).
"La multiprogramación marcó la transición de sistemas batch a interactivos, explotando el paralelismo entre CPU y E/S"
(Silberschatz, Galvin y Gagne, 2018, p. 78).

Tipos: mainframe, servidor, multiprocesador, embebidos y tiempo real

Tipos de Sistemas Operativos

1. Mainframe

Definición:

SO diseñados para mainframes (computadoras de alta capacidad utilizadas en entornos empresariales y gubernamentales).

Características:

  • Alta disponibilidad: Funcionamiento continuo (99.999% uptime).
  • Soporte para miles de usuarios concurrentes.
  • Ejemplos: IBM z/OS, UNIX System Services.
"Los SO para mainframes priorizan la confiabilidad, escalabilidad y seguridad en entornos críticos"
(Silberschatz, Galvin y Gagne, 2018, p. 89).

2. Servidor

Definición:

SO optimizados para gestionar servicios en red (web, correo, bases de datos).

Características:

  • Multiusuario y multitarea avanzada.
  • Soporte para hardware de alto rendimiento (múltiples CPUs, RAID).
  • Ejemplos: Windows Server, Linux (RHEL, Ubuntu Server).
"Los SO servidor deben equilibrar rendimiento, seguridad y capacidad de administración remota"
(Stallings, 2018, p. 112).

3. Multiprocesador (Sistemas Paralelos)

Definición:

SO que gestionan computadoras con múltiples CPUs/núcleos para alto rendimiento.

Características:

  • Compartición de memoria (SMP) o clusters.
  • Planificación equilibrada entre procesadores.
  • Ejemplos: UNIX, Linux (con soporte para NUMA).
"La sincronización entre procesadores es clave para evitar condiciones de carrera y garantizar escalabilidad"
(Tanenbaum & Bos, 2015, p. 67).

4. Embebidos (Embedded)

Definición:

SO para dispositivos con recursos limitados (IoT, electrodomésticos, wearables).

Características:

  • Bajo consumo de energía y memoria.
  • Tiempo real (a menudo RTOS).
  • Ejemplos: FreeRTOS, Embedded Linux, QNX.
"Los SO embebidos sacrifican funciones generales por eficiencia y especialización"
(Laplante, 2020, p. 134).

5. Tiempo Real (RTOS)

Definición:

SO que garantizan respuestas en tiempos deterministas (críticos para industria, medicina, etc.).

Tipos:

  • Hard Real-Time: Fallar en un plazo es inaceptable (ejemplo: control de aviones).
  • Soft Real-Time: Plazos flexibles (ejemplo: streaming multimedia).

Ejemplos:

VxWorks, RTLinux.

"Un RTOS debe priorizar la predictibilidad sobre el rendimiento bruto"
(Liu, 2019, p. 211).
